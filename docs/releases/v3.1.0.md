# v3.1.0 - Docker and Kubernetes Readiness

## Summary

This release adds containerization and Kubernetes readiness to the AWS Employee Management Application.

The existing AWS EC2, SSM, ALB, ASG, and RDS deployment path remains intact. Docker and Kubernetes support is added as an additional local/demo and future production path.

## Added

- `Dockerfile` for PHP 8.2 + Apache application image
- `.dockerignore` to keep build context small
- `docker-compose.yml` with app + MySQL local stack
- Docker health checks
- `docs/containerization.md`
- Kubernetes manifests under `k8s/`
- `docs/kubernetes.md`
- Kubernetes app Deployment with two replicas
- Kubernetes Service, ConfigMap, Secret example, Ingress, and HPA
- Optional local Kubernetes MySQL manifest

## Docker Verification

Run from the repository root:

```bash
docker compose up --build
```

Open:

```text
http://localhost:8080
```

Verify:

- Home page loads
- Health check returns `OK`
- Add Employee works
- Search Employee works
- Edit Employee pre-fills values
- Update Employee works
- Delete Employee works

Stop:

```bash
docker compose down
```

## Kubernetes Verification

Build or load the local image, then apply manifests:

```bash
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/configmap.yaml
kubectl apply -f k8s/secret.example.yaml
kubectl apply -f k8s/mysql-dev.yaml
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
```

Port forward:

```bash
kubectl port-forward -n employee-management service/employee-management-service 8080:80
```

Open:

```text
http://localhost:8080
```

## Production Note

For production Kubernetes, use Amazon RDS or another managed database instead of `k8s/mysql-dev.yaml`.

EKS is intentionally documented as a future option, not required for this release, because EKS can add AWS costs beyond Free Tier usage.
