# v3.2.0 - Add Application Authentication

## Summary

This release adds simple PHP session authentication to the Employee Management Application.

The existing AWS, Docker, and Kubernetes deployment paths remain intact.

## Added

- Login page
- Logout action
- PHP session protection for employee dashboard
- Automatic `users` table creation
- Initial admin user seeding from environment variables
- Password hashing with `password_hash()`
- Password verification with `password_verify()`
- Auth environment variables for Docker Compose
- Auth environment variables for Kubernetes secret example
- Optional GitHub Actions secrets for AWS deployments
- Authentication documentation

## New Environment Variables

| Variable | Purpose |
| --- | --- |
| `AUTH_ADMIN_USER` | Initial admin username |
| `AUTH_ADMIN_PASS` | Initial admin password |

Local demo defaults:

```text
admin
ChangeMe123!
```

Use stronger values before deploying to a shared environment.

## Verification Checklist

- Login page appears before dashboard

- Valid login opens employee dashboard
- Add Employee works after login
- Search Employee works after login
- Edit Employee works after login
- Delete Employee works after login
